Connection Game — Project Summary

Rules

  1. Number of players: N ≥ 3. One player is the clue-giver (A). The remaining N−1 players form a team (B).
  2. A chooses a hidden word W and submits it to the game server. The server stores W privately for A.
  3. A reveals the first letter of W. The known prefix is X (initially the first letter).
  4. Team B has 5 lives for each prefix X. If X extends by 1 letter, lives reset to 5.
  5. Gameplay loop:
    • Any B member (b1) has 60s to give a descriptive hint (phrase/word) that points to a target word Z which starts with the prefix X.
    • Another B member (b2) can press Connection if they think b1's hint points to Z. This triggers a 10s secret-guess window for b2 and A.
    • During the 10s window both b2 and A submit guesses privately. If A correctly identifies Z, B loses one life for the current X.
    • If b2 correctly guessed Z and A did not, connection succeeds and A reveals the next letter of W (X grows by 1). Lives reset to 5.
    • If connection fails and either b1 or b2 happened to guess W exactly then B immediately wins.
    • If B loses all 5 lives on a prefix X (A guessed 5 hints correctly), A wins the round.
  6. End conditions:
    • B wins if full W is revealed or someone explicitly guesses W.
    • A wins if B runs out of 5 lives on the same prefix X.

Files & What they do

server/

  • GameServer.java
    Main server process. Responsibilities:
    • Open a ServerSocket and accept client connections.
    • Spawn a ClientHandler thread for each connected client.
    • Maintain authoritative GameState and broadcast updates to clients.
  • ClientHandler.java
    Per-client worker that:
    • Reads raw messages from the client's socket and converts them into Message objects.
    • Handles client actions (submit-word, hint, press-connection, secret-guess, guess-W) and applies server-side validation.
    • Sends state updates, timer events and reveal messages back to the client.
  • TimerManager.java
    Centralized timer utility used by the server for:
    • 60s hint window (starts when a hint is posted).
    • 10s secret-guess window (after connection press).
    • Per-prefix life tracking and scheduling tasks to trigger life loss / reveal next letter.
  • Message.java
    Server-side protocol object (used when reading raw socket input). Typical fields:
    • type (e.g., HINT, CONNECTION, SECRET_GUESS, SUBMIT_WORD, UPDATE_STATE)
    • from (player id or name)
    • payload (map or JSON string)
  • GameState.java
    Authoritative game model: stores the current prefix X, the hidden word's length, revealed letters, current lives left, player list and roles. Server methods include:
    • revealNextLetter(), decrementLife(), resetLives(), checkWin().

client/

  • GuiClient.java
    Client-side UI implementation (JavaFX in your repo). Responsibilities:
    • Render the game: current prefix X, lives left, hint input, Connection button, chat/log and countdown timers.
    • Handle UI events and call ClientController for network operations.
  • ClientController.java
    Networking layer for the client:
    • Maintain a persistent Socket to the server and send/receive Message JSONs.
    • Expose methods like sendHint(), pressConnection(), submitSecretGuess(), and submitWord() for the UI to call.
  • ParsedMessage.java
    Client-side helper that converts raw Message payloads into richer objects for the UI (helpers like isUpdate(), getRemainingTime()).

Networking

The project uses a classic client–server architecture with TCP sockets. High-level networking notes:

  • Transport: Java ServerSocket on the server and Socket on the client. Each client connection is handled concurrently by a ClientHandler thread.
  • Protocol: Small JSON messages or a lightweight line-based protocol. Message envelope contains type, from and a payload.
  • Authority: Server holds the canonical GameState. Clients only send action requests and the server validates and broadcasts resulting state changes.
  • Timers: Timers run server-side (via TimerManager) to prevent client tampering. The server sends countdown updates for UI display.
  • Secret guesses: Submitted privately to the server; the server holds them until the 10s window ends and then resolves the result.
